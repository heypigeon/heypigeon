viffie - virtual office for remote work

viffie is an application that allows a user or the group of users collaborate, communicate and interact without the spacial borders in the real time.

Market Research

The goal of the market research was to define the potential competitors on the market, their strengths, and weaknesses and to determine the business goals for viffie. At this stage, the Most Viable Product features were proposed herewith the competitor analysis and existing task and management applications comparison and results of the first survey “Working remotely in a virtual office.”

User Research

viffie is an application that focuses on the user needs. Understanding how people work remotely and what they need to success in their jobs was crucial to this project.
Thanks to several interviews with potential users of viffie I have created primary, secondary and complementary target groups along with four User Personas. The next step was creating the user experience map and task modeling which revealed the pain points and critical moments while working remotely. This knowledge opened the new ways to improve viffie MVPs.

Idea Conception

Idea Conception
A great set of data collected during the market and user research was further analyzed, which gave me a clear view of where viffie is heading.
The conceptual phase was conducted with the set of assumptions concluded from the analysis of user needs and business goals.

Finding The right Structure

At this point, I had a clear idea of the business goals and user needs and enough information to start organizing the structure of the content. The project was divided on web page and application.
Information Architecture on web page was created with AARRR technique and considered future marketing and social media campaigns.
The goal of IA set up for application was to create a modular, flexible yet easy to use and efficient software. In order to achieve this goal, the “Sliders” columns idea was presented and the information architecture designed in the way that allows the user accessing every feature from any point.

Giving it Shape
The Lean UX approach, design-validate- iterate, was present during the whole process of creating viffie.
The time for visualizing the ideas and concepts arrived so I created a set of paper prototypes which were tested on a group of users.
The results of those tests influenced the changes implemented during wireframes design.
Having three sets of wireframes (mobile, tablet and desktop) I used three several applications (inVision, Marvell App, atomic.io) and built interactive prototypes.
At this point the user tests were repeated, not efficient elements removed or improved, the concepts revised and new assumptions created.

User Centered Design

User Centered Design
User Centered Design methodology and its variety of research methods were keeping my focus on understanding motivations, goals and needs of the users and allowing to design the most optimal solutions that help the users

to achieve their goals and at the same time
accelerate the business growth.
I conducted several interviews, survey and tests. Tests conducted: card sorting; A/B, preference, click tests; user flows tests with interactive prototypes.

Material Design Lite

The visual design of viffie is based on Material Design principles.
The goal was to create neutral, yet friendly User Interface that matches different tastes. The most important points were to create a user interface that does not distract the user from work and to make sure that the user interactions support the task flows and are implemented in a logical and dynamic way.
